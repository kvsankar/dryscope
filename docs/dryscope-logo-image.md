# dryscope Logo Image Brief

Use this document when creating a dryscope logo image with ChatGPT Images 2.0
or another image model.

## Preamble For The Logo Agent

Create a logo, not an illustration, poster, mascot, or process diagram. Treat
this file as the source of truth for the logo direction.

Return a single finished logo image. Do not output SVG code, Mermaid, ASCII art,
or a textual explanation. The result should work as a long rectangular logo
lockup for a README header, documentation banner, package page, or website
masthead.

This should not look like a poster, hero banner, wallpaper, or astronomy scene.
The background must not carry the design. The logo should still work if placed
on a plain white or transparent background.

## Core Concept

The logo should combine three ideas:

- **DRY**: repeated code/documentation patterns are detected and consolidated.
- **scope**: a telescope or scope is used to inspect a large field.
- **dryscope**: a developer tool that scans large repositories for repeated
  implementation shapes and documentation overlap before AI-assisted cleanup.

The wordmark must read:

```text
DRYscope
```

Use `DRY` in uppercase and `scope` in lowercase. The word must be legible at a
small size. Do not split the product name into unrelated words.

Keep `scope` clearly readable. `DRY` may be stronger, but `scope` must not fade
into the background or become a low-contrast decorative suffix.

## Required Visual Elements

Include all of these:

- A clean wordmark: `DRYscope`
- A small telescope or spotting-scope sketch beside the wordmark
- A visual cue for repeated patterns, such as repeated star groups, matching
  constellations, duplicated points, or repeated geometric/code-like marks
- A sense that the scope is identifying or focusing on those repeated patterns

The repeated-pattern cue should be subtle and logo-like. It should not turn the
logo into a detailed astronomy illustration. It also should not be so faint
that it reads as generic decoration; the repeated pattern idea should be visible
in a few seconds.

## Preferred Composition

Use a long horizontal rectangle. Prefer an aspect ratio around 4:1 or 5:1. Do
not use a tall, square, or poster-like canvas.

Recommended layout:

1. Left or right: compact telescope sketch.
2. Center: `DRYscope` wordmark.
3. Near the telescope or behind/around the wordmark: a small field of repeated
   star/pattern marks, with two or three repeated groups visibly matching.

Good arrangements:

- Telescope on the left, pointed toward repeated star/pattern groups that sit
  above or beside the wordmark.
- Wordmark centered, telescope as a small companion mark on one side.
- `DRY` slightly emphasized, with repeated pattern marks subtly echoing the
  letters or appearing in the telescope's field of view.

Avoid a tall icon-only layout. The final asset should feel like a rectangular
logo lockup, not a square app icon or a cinematic banner.

## Visual Style

Use a precise, modern, technical style:

- clean vector-like drawing
- transparent background if possible; otherwise a plain white or very light
  background
- restrained color palette
- crisp linework
- strong readability
- minimal details
- no photorealism
- no busy starfield
- no decorative sci-fi glow
- no dark vignette, spotlight cone, fog, or gradient-heavy poster background

Suggested palette:

- deep navy or ink for the main wordmark
- cool blue for scope/inspection accents
- muted green or teal for matched/repeated pattern marks
- small amber highlight only if useful for the focus/selection cue

The style should feel like an open-source developer tool logo: polished,
memorable, and simple enough to reproduce.

## Typography Direction

The wordmark should be the hero.

Requirements:

- `DRY` must be uppercase.
- `scope` must be lowercase.
- The wordmark must be readable, not overly abstract.
- Prefer a geometric sans or technical sans feel.
- `DRY` may be slightly heavier, darker, or accented.
- `scope` should remain high contrast and similar enough in weight to read as
  part of the same wordmark.
- Do not use script, serif display, cartoon, or handwritten lettering.

Optional lettering idea:

- `DRY` can be visually associated with deduplication by showing two small
  repeated marks converging into one near the letters.
- The `o` in `scope` can subtly echo a lens, but do not make it so decorative
  that the word becomes hard to read.

## Telescope / Scope Mark

The telescope should be a sketch-like icon, not a detailed instrument.

Good options:

- simple line telescope on a small tripod
- compact spotting scope
- minimal tube-and-lens shape
- a small scope directed toward repeated star/pattern clusters

Do not use:

- a microscope
- binoculars unless the telescope idea remains obvious
- a camera lens alone
- a rocket, planet, astronaut, or space mascot

## Showing The DRY Aspect

The logo should imply detection of repeated patterns.

Preferred approaches:

- Show two or three small constellations with the same shape, and the telescope
  highlights one matching group.
- Make the repeated clusters visibly identical or near-identical, not just
  generic scattered stars.
- Show repeated star clusters connected by faint matching lines.
- Show duplicate code-like marks, such as `{ }`, `<>`, or small line blocks,
  arranged like stars.
- Show repeated pattern groups becoming one highlighted canonical group.

Keep this cue minimal. It should explain the name without overpowering the
wordmark.

Avoid:

- literal water drops or "dry" desert imagery
- cracked earth
- laundry/drying metaphors
- giant duplicated paragraphs
- dense code blocks
- large celestial scenes that make it look like an astronomy product

## Output Requirements

The final image should be:

- a single PNG
- long rectangular format, roughly 4:1 or 5:1 aspect ratio
- suitable for a README header and documentation banner
- clean at small sizes
- readable on a light background
- visually balanced with the telescope beside the wordmark

If transparency is supported, use a transparent background. If not, use a clean
off-white or very light background.

Avoid using a large textured background to create drama. The logo should be
usable as an asset over normal documentation pages.

## Negative Constraints

Do not create:

- a square app icon
- a full-page illustration
- a poster-style banner with a large background effect
- a process diagram
- a dark neon space scene
- a dark vignette or spotlight-heavy composition
- a photorealistic telescope
- a mascot
- a logo where `DRYscope` is misspelled or split incorrectly
- a logo where `DRY` is lowercase
- a logo where the telescope replaces the wordmark
